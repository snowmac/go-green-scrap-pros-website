This week — route cards (Mon–Fri)

LIP runs on the customer's regular trash day inside the active district. If a Monday observed federal holiday falls in the week, the entire week's trash collection shifts one day later (Mon→Tue, Tue→Wed, etc.) and Friday rolls to Saturday.

9-week rotation (anchored)

Anchor: May 18–22, 2026 = District 3 (per 2026 Week-B WasteWise). Rotation projects ±8 weeks from this week.

PoC architecture — SCRAP-MAP

  • Denver 9-week rotation: deterministic district = ((weekIndex - anchorWeekIndex) mod 9) starting from D3 anchor.
  • Static GIS polygons: ship district boundary GeoJSON as bundled static assets — no DB, no live API.
  • Seed-address polling: verify each district by spot-checking ~3 seed addresses against Denvergov.org/TrashSchedule before each on-route shift.
  • Alley-vs-curb prioritization: tag each block by collection side (alley/curb); route alley-loaded blocks first for visibility.
  • Holiday buffer: when a federal Monday holiday falls in-week, shift Mon→Tue and roll Fri→Sat; flag the affected week explicitly.

Frontend

Single-file static page, no JS framework, no storage APIs. Computes the active week and district at load time in America/Denver.
